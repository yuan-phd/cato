## 2. Findings

### Praise

**Praise — `test-run-2/count_words.py:28-31`**
The `TypeError` check uses `isinstance(text, str)` rather than `type(text) is
str`, which correctly accepts `str` subclasses if any exist while still
rejecting `bytes`, `None`, `int`, etc. The error message includes the actual
received type name (`type(text).__name__!r`), which makes debugging a misuse
of the API much easier than a bare `raise TypeError`.

**Praise — `test-run-2/count_words.py:33-38`**
The body of the function is six lines and reads exactly as the spec phrases
the algorithm. Using `Counter` for accumulation and `dict(counter)` for the
return value is idiomatic and matches the spec's note that internal `Counter`
use is fine but the public return type must be plain `dict`.

**Praise — `test-run-2/test_count_words.py`**
Tests are mapped 1:1 to the spec's 20-item list with numbered comments above
each function, making spec-to-test traceability obvious. The "entirely
punctuation token" test (line 93) goes beyond the equality assertion and
explicitly checks `"" not in result` and `"!!!" not in result`, which is the
strongest form of the assertion the spec requested. Splitting test 20 into
two functions (`test_type_error_on_int` and `test_type_error_on_bytes`) is a
small improvement over the spec's single bullet — failures will identify
which input variant broke.

### Nit

**Nit — `test-run-2/count_words.py:7`**
Imports the whole `collections` module and uses `collections.Counter`. A more
common Python convention is `from collections import Counter` and then
`Counter()` at the call site. This is purely stylistic — both forms are
correct and the current form is arguably more explicit. Author's choice; do
not change unless preferred.

**Nit — `test-run-2/test_count_words.py:8`**
`from count_words import count_words` works because pytest's rootdir/conftest
behavior adds the test file's directory to `sys.path` during collection. This
is fine for the gitignored, ephemeral `test-run-2/` directory, but if this
code ever migrates into a package, the import will need to become
`from test_run_2.count_words import count_words` (or similar). The spec
explicitly disclaims packaging concerns, so this is purely informational.
